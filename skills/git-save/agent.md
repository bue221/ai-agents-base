# git-save

## Description
Create safe, conventional commits with checks and summaries.

## When to use
- Before committing changes.
- When you need consistent commit messages.

## Steps
1) Review git status, diff, and recent log.
2) Summarize changes and draft a conventional message.
3) Stage only relevant files (avoid secrets).
4) Commit and verify status.

## Output
- Commit message draft + safety notes.

## Notes
- Never force-push main.
