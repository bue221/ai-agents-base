# Guía de Contribución: ai-agents-base

¡Qué nota que quiera aportar, parce! Aquí le explico cómo lo hacemos mka.

## 📂 Dónde meter las cosas

1. **Reglas Globales:** Si es algo que afecta a todos los agentes, va en `agents.md`.
2. **Skills:** Si es una habilidad específica (ej. "Refactorizador de React"), cree una carpeta en `skills/mi-skill/` con un archivo `SKILL.md`.
3. **Subagentes:** Si es un rol especializado, métalo en `subagents/mi-rol.md`.

## 📝 Estándares de Código

- **Idioma:** Usamos español para las explicaciones, pero los términos técnicos (e.g., *sinks*, *hooks*, *context*) siempre en **English** mka.
- **Formato:** Markdown estándar. Use bloques de código para los ejemplos.
- **Personalidad:** Mantener el tono de "Mentor Técnico Experto". Queremos ser útiles pero con rigurosidad profesional.

## 🚀 Proceso de Pull Request

1. Haga un fork del repo.
2. Cree una rama para su cambio: `git checkout -b feat/mi-nuevo-skill`.
3. Suba sus cambios y mande el PR.
4. Espere a que un "parce" lo revise mka.

¡Hágale de una!
